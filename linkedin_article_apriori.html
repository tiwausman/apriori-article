Have you ever wondered how Amazon knows what to suggest next, or why supermarkets place certain products next to each other? The answer is Market Basket Analysis — and the engine behind it is the Apriori algorithm.

1The Dataset

We work with the UCI Online Retail Dataset — a real-world transactional dataset from a UK-based e-commerce retailer covering December 2010 to December 2011. Each row is one product in one invoice.

Python · Step 1 — Import libraries
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from mlxtend.frequent_patterns import apriori, association_rules

# Load the dataset
df = pd.read_excel('Online Retail.xlsx')
display(df.head())
print(df.shape)      # (541909, 8)
Dataset structure

Each record contains: InvoiceNo, StockCode, Description, Quantity, InvoiceDate, UnitPrice, CustomerID, Country. One invoice can span many rows — one row per product purchased.

2Data Cleaning

Real-world retail data is messy. Before running any algorithm, we must remove invalid records that would distort our association rules.

Python · Step 2 — Clean the data
# Remove rows with missing product descriptions
df = df.dropna(subset=['Description'])

# Drop CustomerID column (not needed for Apriori)
df = df.drop(columns=['CustomerID'])

# Remove cancelled transactions (InvoiceNo starts with 'C')
df = df[~df['InvoiceNo'].astype(str).str.startswith('C')]

# Remove returns and invalid quantities
df = df[df['Quantity'] > 0]

# Drop duplicates
df.drop_duplicates(inplace=True)

print(df.shape)  # (525462, 7)  ← 16,447 records removed
Why keep rows with missing CustomerID? Dropping them would remove ~135,000 valid product transactions. Since CustomerID is not required for Apriori (which groups by InvoiceNo), we keep these rows and only remove those missing a product Description.

4Building the Basket Matrix

The Apriori algorithm needs a binary matrix — one row per invoice, one column per product, with 1 meaning "purchased" and 0 meaning "not purchased".

Python · Step 4 — Basket transformation
# Filter to United Kingdom (highest transaction volume)
basket = (df[df['Country'] == 'United Kingdom']
          .groupby(['InvoiceNo', 'Description'])['Quantity']
          .sum()
          .unstack()
          .fillna(0))

# One-hot encode: 1 = purchased, 0 = not purchased
def hot_encode(x):
    return 0 if x <= 0 else 1

basket = basket.applymap(hot_encode)

# Convert to Boolean for memory efficiency
basket = basket > 0

print(basket.shape)  # (rows=invoices, cols=unique products)
Why United Kingdom? The UK accounts for the majority of transactions in the dataset. Using one region avoids mixing purchasing patterns from different countries, which would dilute the association signals.

5Running the Apriori Algorithm

Due to memory constraints (the full basket requires ~11.8 GiB), we apply practical optimisations while preserving meaningful results.

Python · Step 5 — Apriori with memory optimisation
from mlxtend.frequent_patterns import apriori, association_rules

# Run Apriori with low-memory mode and max pair size
frequent_items = apriori(
    basket,
    min_support=0.005,     # 0.5% of invoices
    use_colnames=True,
    low_memory=True,       # avoids MemoryError
    max_len=2              # pairs only (1 and 2-itemsets)
)

# Generate association rules with ≥ 50% confidence
rules = association_rules(
    frequent_items,
    metric='confidence',
    min_threshold=0.5
)

print("Frequent itemsets:", len(frequent_items))
print("Association rules:", len(rules))

6The Three Metrics That Matter

Support
Frequency
How often the itemset appears across all transactions. Filters out rare combinations.
Confidence
Reliability
P(B|A) — if a customer buys A, how likely are they to buy B?
Lift
Strength
> 1 = positive link. How much stronger than random chance is this association?
Python · Step 6 — Analyse and filter rules
# Top 20 rules sorted by Lift
top_rules = rules[['antecedents', 'consequents',
                    'support', 'confidence', 'lift']]\
            .sort_values(by='lift', ascending=False)

# Convert frozensets to readable text
rules_display = rules.copy()
rules_display['Antecedent'] = rules_display['antecedents'].apply(
    lambda x: ', '.join(list(x)))
rules_display['Consequent'] = rules_display['consequents'].apply(
    lambda x: ', '.join(list(x)))

# Auto-interpret top 10 rules
top10 = rules_display.sort_values(by='lift', ascending=False).head(10)
for _, row in top10.iterrows():
    print(f"If a customer buys '{row['Antecedent']}',")
    print(f"they are likely to buy '{row['Consequent']}'.")
    print(f"Confidence: {row['confidence']:.2%} | Lift: {row['lift']:.2f}")
    print("-" * 60)

Support distribution
Most rules have support below 2% — rare combinations but very reliable when they occur. A few high-support rules represent the most popular co-purchases.
Confidence range
50%–75% confidence for most rules — these associations are reliable predictors, making them ideal for product recommendation engines.
Seasonal context
November and December generate the most transactions, yielding the most reliable frequent itemsets — perfect for holiday campaign planning.
UK market focus
UK dominates with 90%+ of transactions. Association rules derived here are most applicable to UK shopping behaviour — other markets may differ.
